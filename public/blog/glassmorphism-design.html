1. What is Glassmorphism

Glassmorphism is a user interface style that mimics frosted glass by combining translucent layers with background blur and soft lighting. Instead of using flat blocks, it builds visual hierarchy with depth, making components feel suspended above colorful atmospheres.

  • Transparency: cards usually use 6% to 16% white opacity.
  • Backdrop blur: 12px to 30px depending on viewport and context density.
  • Subtle borders: thin 1px lines around cards to define edges.
  • Layer depth: shadows, glow accents, and spatial offsets create stack order.

5. How to Use It

Design glassmorphism as a system, not a one-off effect. The key is consistency across opacity, blur, border, elevation, and fallback behavior.

  • Apply one blur scale for base cards and one for elevated modals.
  • Reserve strongest glow for interactive priorities like CTA and active tabs.
  • Increase overlay darkness behind text-heavy glass cards for readability.
  • Keep motion subtle: depth shifts, parallax, and reveal blur transitions are enough.

6. Accessibility Issues (Important)

Glass UIs can quickly fail accessibility when decorative depth overpowers legibility. Contrast ratio, text background stability, and focus visibility must be enforced.

  • Contrast issues: low-opacity cards over noisy backgrounds reduce text clarity.
  • Readability problems: small text on bright blur layers loses edge definition.
  • Fixes: stronger overlays, reduced blur behind text blocks, and high-contrast text tokens.

Tokenize your glassmorphism system so teams can ship consistent interfaces at scale. These token ranges are production-safe for dashboards and product pages.

Blur Tokens

--blur-sm: 12px
--blur-md: 20px
--blur-lg: 30px

Elevation Tokens
